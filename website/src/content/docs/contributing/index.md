Cardano Dev Skills is community-curated. The contribution surface is small
and well-documented — pick the page that matches what you want to do.

- **[Scope](/cardano-dev-skills/contributing/scope/)** — what's in, what's out,
  and the borderline rule. Read this first if you're considering adding a
  source for a specific dApp or product.
- **[Add a source](/cardano-dev-skills/contributing/add-a-source/)** — the
  vetting bar, the schema, validation, and how to test a fetch locally.
- **[Add a skill](/cardano-dev-skills/contributing/add-a-skill/)** — the
  skill format, the quality checklist, and the validation steps.
- **[Governance](/cardano-dev-skills/contributing/governance/)** — the
  doc-update checklist, the auto-derived count mechanism, and the source
  maintenance bar.

The canonical contribution guide lives in
[`docs/CONTRIBUTING.md`](https://github.com/easy1staking-com/cardano-dev-skills/blob/main/docs/CONTRIBUTING.md)
in the repo. These pages mirror it; the repo file is authoritative if they
ever drift.

For freeform feedback or to suggest a missing topic, use the
[issue templates](https://github.com/easy1staking-com/cardano-dev-skills/issues/new/choose).
